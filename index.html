This static site reads a pre-built JSON file at /data/boston_advantage_2012.json which is created by the GitHub Actions workflow `fetch-schedule.yml` (see below). If the JSON is not present, this script will show a friendly message. */ async function loadData(){ const winsEl = document.getElementById('wins'); const lossesEl = document.getElementById('losses'); const tiesEl = document.getElementById('ties'); const tableBody = document.querySelector('#schedule-table tbody'); try{ const res = await fetch('/data/boston_advantage_2012.json', {cache: 'no-store'}); if(!res.ok) throw new Error('No data file'); const data = await res.json(); winsEl.textContent = data.summary.wins; lossesEl.textContent = data.summary.losses; tiesEl.textContent = data.summary.ties; tableBody.innerHTML = ''; if(data.games && data.games.length){ data.games.forEach(g=>{ const tr = document.createElement('tr'); tr.innerHTML = `${g.date || ''}${g.opponent || ''}${g.location || ''}${g.time || ''}${g.result || ''}`; tableBody.appendChild(tr); }) } else { tableBody.innerHTML = 'No upcoming games found.'; } }catch(err){ tableBody.innerHTML = 'Schedule data not found. Edge - Upper. W · 3-1. Completed" # We'll look back and forward few lines to assemble a full record. idx = lines.index(line) window = ' '.join(lines[max(0, idx-3): idx+4]) # date regex date_match = re.search(r'([A-Za-z]{3,9}\s*\d{1,2}/\d{1,2}|[A-Za-z]{3,9}\s+\d{1,2})', window) time_match = re.search(r'(\d{1,2}:\d{2}\s*(?:AM|PM|am|pm)?)', window) result_match = re.search(r'([WL]\s*·\s*\d+-\d+|W\s*·\s*\d+-\d+|L\s*·\s*\d+-\d+|T\s*·\s*\d+-\d+|TIE|Tie)', window) opponent_match = None # opponent often appears as a team name after the time or rink; capture a fragment that contains a dash or 'vs' or similar opp = '' m = re.search(r'(vs\.?\s*[^.\n]+|v\.?\s*[^.\n]+|\.\s*([^\d]+?)\.|-\s*([^\d]+?)\.)', window) if m: opp = (m.group(0)).strip() g = { 'raw': window, 'date': date_match.group(0) if date_match else '', 'time': time_match.group(0) if time_match else '', 'opponent': opp, 'result': result_match.group(0).replace('·','').strip() if result_match else '' } games.append(g) # Basic win/loss/tie counts from results wins = sum(1 for g in games if g.get('result','').upper().startswith('W')) losses = sum(1 for g in games if g.get('result','').upper().startswith('L')) ties = sum(1 for g in games if g.get('result','').upper().startswith('T')) out = { 'source': URL, 'fetched_at': datetime.utcnow().isoformat()+'Z', 'summary': {'wins': wins, 'losses': losses, 'ties': ties}, 'games': games } import os os.makedirs(os.path.dirname(OUT_PATH), exist_ok=True) with open(OUT_PATH, 'w') as f: json.dump(out, f, indent=2) print('Wrote', OUT_PATH) ``` --- ### `.github/workflows/fetch-schedule.yml` ```yaml name: Fetch Elite9 schedule on: workflow_dispatch: schedule: - cron: '0 8 * * *' # daily fetch at 08:00 UTC (optional) jobs: fetch: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - name: Set up Python uses: actions/setup-python@v4 with: python-version: '3.11' - name: Install deps run: | python -m pip install --upgrade pip pip install requests beautifulsoup4 - name: Run fetch script run: | python scripts/fetch_schedule.py - name: Commit generated data run: | git config user.name "github-actions[bot]" git config user.email "41898282+github-actions[bot]@users.noreply.github.com" git add docs/data/boston_advantage_2012.json || true git commit -m "Update Boston Advantage 2012 schedule data" || echo "no changes" git push env: GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }} ``` --- ## How this works 1. The GitHub Action fetches the Elite 9 schedule page and runs a small Python parsing script that extracts lines mentioning **Boston Advantage** and outputs a JSON file at `docs/data/boston_advantage_2012.json`. 2. GitHub Pages serves the site from the `docs/` folder. The static site (`index.html`) fetches the generated JSON and displays the win/loss/tie counts plus the upcoming games. * The Elite 9 site uses different templates and sometimes dynamic content; the parser is intentionally conservative (heuristic). It should catch many schedule lines but may need tuning if the site structure changes. If you want a more robust parser I can improve it to target the exact HTML elements (but that requires confirming the exact team page structure). * If you prefer client-only (no workflow), I can provide an approach using a public CORS proxy — but build-time fetching (the workflow) is more reliable and cleaner for GitHub Pages. --- ## What you need to do 1.
